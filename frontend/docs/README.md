# Frontend 기획 변경 문서

프론트엔드 UI 기획 변경 사항과 그 근거를 보존하는 디렉토리.

코드 자체로는 "왜 이렇게 만들었는지"가 드러나지 않는 결정들 — 식별자 포맷, 표시 컨벤션, 레이아웃 우선순위 결정 등 — 을 여기 기록한다.

## 문서 목록

- [raw-data-title-naming.md](./raw-data-title-naming.md) — Raw Data 파일명(타이틀) 네이밍 컨벤션 (Sentinel-1 차용)

## 작성 가이드

- 한 결정 = 한 문서. 한 파일에 여러 변경을 섞지 않는다.
- 파일명: `kebab-case.md`.
- 문서 상단에 다음 메타를 둔다:
  - **결정 일자**
  - **상태** (Active / Superseded by `<other.md>` / Deprecated)
  - **관련 코드** (영향받는 파일 경로 또는 함수)
- 본문은 다음 순서를 권장:
  1. 배경 / 문제
  2. 결정
  3. 대안과 트레이드오프
  4. 적용 범위 / 영향
  5. 미해결 사항
