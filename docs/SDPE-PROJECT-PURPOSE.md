# SDPE 프로젝트 목적

## 개요

SDPE(SAR Data Processing Element)는 SAR 위성 원시 데이터를 운영 환경에서 안정적으로 처리해 최종 SAR 제품으로 만들고, 그 과정을 운영자가 관리·감시·재처리할 수 있게 하기 위해 만들어진 프로젝트이다.

핵심 문제는 복잡하고 오래 걸리며 실패 가능성이 있는 SAR 처리 파이프라인을 운영자가 안전하게 구성, 실행, 추적, 복구할 수 있게 하는 것이다.

## 해결하려는 문제

### 1. 원시 SAR 데이터는 바로 사용할 수 없다

위성에서 받은 원시 데이터는 패킷 복원, BAQ 압축 해제, 레이더 라인 재구성, 보정, HDF5 변환 같은 Level-0 처리가 필요하다.

이후 Range/Azimuth 압축, SLC/GRD/GEC/MAP 생성, DEM 보정 등을 거쳐야 실제 영상 제품으로 활용할 수 있다.

관련 근거:

- `interfaces/csc-3/README.md`
- `interfaces/csc-4/README.md`

### 2. 처리 과정이 길고 단계 간 의존성이 있다

SAR 처리는 다음과 같은 순서 있는 다단계 파이프라인이다.

```text
수신 -> L0 -> L1 -> L2 -> L3 -> 카탈로그 등록 -> 배포
```

각 단계의 산출물이 다음 단계의 입력이 되므로, 단순 스크립트가 아니라 작업 할당, 완료 이벤트 수신, 다음 단계 트리거가 필요하다.

관련 근거:

- `docs/screens/DAG-RATIONALE.md`
- `interfaces/csc-8/README.md`

### 3. 실패, 지연, 품질 문제에 운영자가 대응해야 한다

운영 환경에서는 처리 실패, 처리 지연, 품질 미달, 큐 적체 같은 상황이 발생할 수 있다.

SDPE는 다음 기능을 통해 이 문제를 다룬다.

- 처리 실패 시 자동 재시도
- 최대 재시도 초과 시 알림 발행
- Dead Letter 확인
- Job 전체 재처리 및 부분 재처리
- 실행 로그와 감사 로그 조회
- 큐 상태 및 처리량 모니터링

관련 근거:

- `docs/usecases/USECASE-REPORT.md`

### 4. 모든 처리를 처음부터 다시 돌리면 비효율적이다

특정 제품이나 특정 처리 Level에서 문제가 생긴 경우, Level-0부터 전체 재처리하는 방식은 비효율적이다.

SDPE는 `LEVEL_1`, `LEVEL_2`, `LEVEL_3` 등 특정 지점부터 다시 처리할 수 있도록 target level 기반 부분 재처리와 DAG 기반 파이프라인을 설계한다.

관련 근거:

- `docs/screens/DAG-RATIONALE.md`
- `interfaces/csc-8/README.md`

### 5. 위성 및 촬영 모드별 처리 설정을 유연하게 바꿔야 한다

LumirX-1, LumirX-2, LumirX-3 같은 위성과 Stripmap, ScanSAR, Spotlight 같은 촬영 모드에 따라 처리 설정이 달라진다.

SDPE는 새 위성이나 촬영 모드가 추가될 때 코드 변경보다 처리 프로파일과 파이프라인 구성을 통해 대응할 수 있도록 설계된다.

관련 근거:

- `docs/usecases/USECASE.md`

### 6. DAG는 이벤트를 수집하지 않고, 이벤트가 들어왔을 때 실행할 파이프라인을 결정한다

원시 데이터 수집 자체는 데이터 수집 백엔드가 담당한다. 예를 들어 수신 이벤트는 백엔드 서버가 pgmq의 `sdpe.reception.events` 같은 큐에서 이벤트 값을 받아 시작하므로, DAG 화면이 원시 데이터 수집을 직접 수행할 수는 없다.

DAG에서 필요한 작업은 다음과 같다.

- 어떤 이벤트 유형이 들어왔을 때 어떤 파이프라인을 실행할지 정의한다.
- 위성, 촬영 모드, 편파, 입력 레벨 같은 이벤트 조건과 파이프라인을 매핑한다.
- 배포된 파이프라인만 자동 실행 대상이 되도록 상태를 관리한다.
- 백엔드 서버가 pgmq 이벤트를 수신하면 이 배포 규칙을 조회해 해당 파이프라인을 자동 기동할 수 있게 한다.

따라서 운영자가 봐야 하는 유즈케이스는 "이 파이프라인을 배포하면 실제 운영에서 이 이벤트 조건에 매칭될 때 자동 실행된다"이다. DAG 화면은 그래프 편집뿐 아니라, 선택한 파이프라인의 자동 실행 규칙과 배포 상태를 함께 보여줘야 한다.

## 정리

SDPE는 SAR 위성 데이터의 수신부터 Level-0/1/2/3 처리, 제품 등록, 모니터링, 실패 대응, 재처리까지 이어지는 운영용 데이터 처리 플랫폼이다.

즉, SDPE는 SAR 처리 알고리즘만을 구현하는 프로젝트가 아니라, SAR 데이터 처리 업무 전체를 안정적으로 운영하기 위한 파이프라인 오케스트레이션 및 운영 관리 시스템이다.
