# UI 스펙 — 처리 프로파일 관리

> [USECASE.md](./USECASE.md)의 **UC17–UC20**을 담당합니다.

## 라우트

`/plan/profiles` · `/current/profiles`

## 목적

위성별·모드별 재사용 가능한 처리 프로파일(처리 파라미터 조합)을 관리한다.
프로파일은 JOB_INIT(CSU-08.02) 단계에서 자동 선택에 사용되며,
다중 위성 확장성(REQ-SCALE-002)에 필수적이다.

---

## 페이지 레이아웃

### 1. 헤더

| 요소 | 설명 |
| --- | --- |
| 페이지 타이틀 | "처리 프로파일" |
| **+ 새 프로파일** 버튼 | 생성 다이얼로그 오픈 (Admin 전용) |
| 필터 컨트롤 | 위성 드롭다운, 모드 드롭다운, 키워드 검색 |

### 2. 프로파일 목록 테이블 (UC17)

| 컬럼 | 설명 |
| --- | --- |
| 이름 | 프로파일 표시명 (상세 링크) |
| 위성 | Lumir-X1 / X2 / X3 |
| 모드 | SM / SC / SL |
| 편파 | HH, VV, HH+HV, VV+VH, Quad |
| 우선순위 | 1(최고) ~ 10(최저) |
| 참조 파이프라인 수 | 해당 프로파일을 사용 중인 파이프라인 수 |
| 생성일 | 타임스탬프 |
| 작업 | 수정 / 삭제 (Admin 전용) |

- 기본 정렬: 위성 → 모드 → 우선순위 오름차순
- 페이지네이션: 서버 측, 20행/페이지

### 3. 생성 / 수정 다이얼로그 (UC18, UC19)

**트리거**: "+ 새 프로파일" 버튼(생성) 또는 "수정" 액션(수정)

| 필드 | 타입 | 검증 |
| --- | --- | --- |
| 이름 | 텍스트 입력 | 필수, 최대 100자, 위성+모드 조합 내 중복 불가 |
| 위성 | 드롭다운 | 필수 (Lumir-X1, X2, X3) |
| 모드 | 드롭다운 | 필수 (SM, SC, SL) |
| 편파 | 다중 선택 | 1개 이상 필수 |
| 우선순위 | 숫자 입력 | 1~10, 기본값 5 |
| 처리 파라미터 | JSON 에디터 / key-value 폼 | 스키마 검증 |
| 설명 | 텍스트 영역 | 선택, 최대 500자 |

**액션**: 저장 / 취소
저장 시 → 목록 갱신, 성공 토스트 표시

### 4. 삭제 확인 (UC20)

**트리거**: 프로파일 행의 "삭제" 액션

- 1개 이상의 파이프라인이 해당 프로파일을 참조 중이라면 **차단 경고** 표시:
  > "이 프로파일은 N개의 파이프라인에서 참조 중입니다. 삭제 전에 먼저 해제하거나 재할당하세요."
- 참조가 없는 경우 일반 확인 다이얼로그 표시:
  > "'{이름}' 프로파일을 삭제하시겠습니까? 이 작업은 되돌릴 수 없습니다."

## 예상 API 엔드포인트

| 동작 | 메서드 | 경로 |
| --- | --- | --- |
| 프로파일 목록 조회 | GET | `/v1/processing-profiles` |
| 프로파일 생성 | POST | `/v1/processing-profiles` |
| 프로파일 상세 조회 | GET | `/v1/processing-profiles/{id}` |
| 프로파일 수정 | PUT | `/v1/processing-profiles/{id}` |
| 프로파일 삭제 | DELETE | `/v1/processing-profiles/{id}` |
