# SDPE Frontend — 유즈케이스 정의서

## 1. 사용자 정의

| 항목      | 내용                                                                            |
| --------- | ------------------------------------------------------------------------------- |
| 역할      | **Administrator** — 전체 기능 접근 (시스템 설정, 감사 로그, 프로파일 관리, 사용자·계정 관리 포함) |
|           | **Operator** — 파이프라인 조회·실행·재처리, Job 관리, 모니터링, 본인 계정 관리  |
| 접근 권한 | Operator 이상 역할 필수. Analyst 이하 접근 불가 (SAD 14.2, REQ-SEC-002)         |
| 책임      | SAR 데이터 처리 파이프라인의 구성·운영·모니터링·장애 대응                       |

> 현재 시스템은 사내 운영팀 전용이며, JWT 기반 인증 + RBAC 역할 분리로 접근을 통제한다 (AD-08).

## 3. 도메인 용어

| 용어           | 설명                                                                                                 |
| -------------- | ---------------------------------------------------------------------------------------------------- |
| 파이프라인     | SAR 데이터 처리 워크플로우를 정의하는 DAG (Directed Acyclic Graph)                                   |
| Job            | 파이프라인의 1회 실행 인스턴스                                                                       |
| 노드 (Step)    | 파이프라인을 구성하는 개별 처리 단계                                                                 |
| TRIGGER        | 원시 데이터 수신 이벤트 (EI-01), 전체 처리의 시작점                                                  |
| FILE_INPUT     | 부분 재처리용 입력 노드 (SI-07), L1/L2부터 재시작                                                    |
| JOB_INIT       | Job 생성 및 처리 프로파일 할당 단계 (CSU-08.02)                                                      |
| SAR            | SAR 처리 단계 — L0, L1A, L1B, L1C, L2A, L2B, L3                                                      |
| CATALOG        | 메타데이터 추출 및 카탈로그 등록 (CSC-07)                                                            |
| 처리 프로파일  | 위성·모드별 재사용 가능한 처리 설정. 위성 추가 시 프로파일만 등록하면 자동 처리 가능 (REQ-SCALE-002) |
| 제품 (Product) | 처리 완료된 SAR 영상 파일. STAC 카탈로그에 등록되어 서비스됨                                         |
| target_level   | 부분 재처리 시 처리 시작 레벨. LEVEL_0~LEVEL_3 (ICD SI-04)                                           |
| trigger_source | 작업 할당 출처. PIPELINE_AUTO / MANUAL_REQUEST / PARTIAL_REPROCESS (ICD SI-04)                       |
| 배포 파이프라인 | 운영 이벤트 자동 실행 대상으로 활성화된 파이프라인. pgmq 이벤트 조건과 매칭될 때 백엔드가 자동 기동한다 |
| 배포 규칙      | 이벤트 유형, 큐, 위성·모드·편파·입력 레벨 조건과 파이프라인을 연결하는 운영 매핑. 동일 큐에 여러 규칙이 있어도 이벤트 1건은 조건에 맞는 규칙 1개로 라우팅된다 |
| Dead Letter    | 최대 재시도 횟수(3회) 초과로 처리 실패한 큐 메시지 (REQ-AVAIL-002)                                   |
| 바이패스       | 노드를 비활성화하여 실행 시 건너뛰는 설정 ※ ICD/SAD 미정의                                           |
| Access Token   | 로그인 시 발급되는 단기 JWT. API 요청 인증에 사용된다 (AD-08)                                        |
| Refresh Token  | Access Token 재발급용 장기 토큰. 유출 시 서버 측 폐기가 가능하다                                     |
| 역할 (Role)    | RBAC 기반 권한 그룹. Administrator / Operator 로 구분된다 (SAD 14.2)                                 |
